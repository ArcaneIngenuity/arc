Detailed Description

A central point from which all Apps may be managed / updated.

A single timing mechanism (loop or callback) is used to run a Hub, from which each App's App_update will be called via Hub_update. This provides implicit synchronisation and best performance across multiple Apps that may be running side-by-side. As such, Hub needs no start / stop like App; Hub_update is run perpetually until the executable terminates.
