Detailed Description

Handles specific game / business / simulation logic within an App.

For every discrete Ctrl needed, allocate it and set the appropriate callbacks.
A Ctrl instance may then be added to an App by setting App.ctrl; this will be the root Ctrl. Often, this will be the only Ctrl needed; though in some cases the user may prefer to use sub-Ctrls attached to root.
Sub-Ctrls may be accessed through the App's root Ctrl.
A Ctrl operates on a model. In arc, models need no specific type; they can be anything (and are internally denoted as void *). Ctrl writes new model state by reading extant model state, inputs and View state.
